- apache: este servidor web de código abierto es el más conocido y usado mundialmente junto a IIS
			 de Microsoft. Aún así, no es el más indicado para apliaciones web pesadas o similares.
			 Esto sumado al apoyo de la comunidad lo hace perfecto para páginas webs de particulares 
			 sin mucha carga o visitas.

- nginx: este servidor está indicado para aplicaciones web de altas prestaciones, haciéndolo una
			opción perfecta para páginas web muy transitadas como Github o el servicio de streaming 
			Netflix.

- thttpd: este servidor web se caracteriza por ser liviano, simple y portatil. Su bajo consumo de
			 recursos lo hace perfecto para servir grandes volúmenes de información estática, tales 
			 como un servidor hosting de imágenes. Un ejemplo de uso puede ser el servidor de 
			 descarga de Napster.

- cherokee: este servidor web español está diseñado para ser muy fácil de administrar, por lo 
				tanto podría decirse que está optimizado para que gente sin mucho conocimiento 
				técnico sea capaz de administrar su propia página web.

- node.js: este es un entorno de aplicación del lado del servidor basado en javascript que puede
			  bajo cualquier servidor web como apache o por sí solo. Basicamente, su cometido sería 
			  hacer correr el código javascript del lado del servidor y no del navegador del cliente.
			  Esto lo hace bastante popular para la creación de aplicaciones en la nube.
